# Larry Academy V2 测试指南

本文档提供完整的测试步骤，确保在部署到生产环境前，所有功能都正常工作。

## 📋 测试前准备

### 1. 环境配置检查

确保 `.env` 文件包含所有必需的环境变量：

```bash
# 检查环境变量文件
cat .env

# 必需的变量
DATABASE_URL=           # ✓ 已配置
NEXTAUTH_URL=          # ✓ 已配置
NEXTAUTH_SECRET=       # ✓ 已配置
OPENAI_API_KEY=        # ✓ 已配置（用于AI游戏生成）
STRIPE_SECRET_KEY=     # ✓ 已配置（测试密钥即可）
```

### 2. 数据库设置

```bash
# 生成Prisma客户端
npm run db:generate

# 运行数据库迁移
npm run db:migrate

# 填充测试数据
npm run db:seed
```

**验证数据库**：
```bash
# 打开Prisma Studio查看数据
npm run db:studio
```

应该看到：
- ✅ 2个测试用户（demo@larryacademy.com, premium@larryacademy.com）
- ✅ 3个示例课程
- ✅ 3个示例游戏
- ✅ 3个示例商品

### 3. 启动开发服务器

```bash
npm run dev
```

访问 http://localhost:3000

## 🐛 常见问题排查

### 数据库连接失败
```bash
# 检查PostgreSQL是否运行
# 检查DATABASE_URL格式
# 尝试重新连接
npm run db:generate
```

### 页面加载错误
```bash
# 清除缓存并重新构建
rm -rf .next
npm run dev
```

### API调用失败
- 检查浏览器Console的错误信息
- 检查Network标签的请求详情
- 确认环境变量配置正确

### Prisma错误
```bash
# 重新生成客户端
npm run db:generate

# 重置数据库（会清空数据！）
npx prisma migrate reset
npm run db:seed
```

## 🚀 测试通过后的部署步骤

测试全部通过后，执行以下步骤进行部署：

### 1. 代码检查
```bash
npm run lint
npm run build
```

### 2. 准备生产环境变量
- 将所有 `_test_` 测试密钥替换为生产密钥
- 确保 `NEXTAUTH_SECRET` 使用强密钥
- 配置生产数据库

### 3. 部署到Vercel（推荐）
```bash
# 推送到GitHub
git add .
git commit -m "Ready for production"
git push origin main

# 在Vercel中导入项目
# 配置环境变量
# 点击Deploy
```

### 4. 部署后验证
- [ ] 访问生产URL
- [ ] 测试注册/登录
- [ ] 检查数据库连接
- [ ] 验证支付webhook
- [ ] 测试AI功能

### 5. 监控和维护
- 设置错误监控（Sentry）
- 配置日志查看
- 定期备份数据库
